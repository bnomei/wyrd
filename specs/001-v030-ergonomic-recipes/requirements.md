# Requirements

R001: WHEN a fixed graph is authored, THE SYSTEM SHALL support validated declarative `weave!` topology throughout cookbook and Bevy examples.

R002: WHEN a reusable graph fragment is authored, THE SYSTEM SHALL support a validated `pattern!` macro with named inputs and outputs.

R003: WHEN a generic host binds a recipe, THE SYSTEM SHALL provide typed, reusable resolved ports and contextual resolution failures.

R004: WHEN examples run scripted frames, THE SYSTEM SHALL provide typed scenario setup and expectations without changing current recipe behavior.

R005: WHEN applications generate graphs, THE SYSTEM SHALL provide scoped composition with domain-typed wires and preserve full catalog access.

R006: WHEN tooling inspects recipes, THE SYSTEM SHALL provide deterministic manifests and feature-gated JSON Schema without changing default/no_std dependencies.

R007: WHEN Bevy installs a recipe, THE SYSTEM SHALL expose typed instance ports while preserving game-owned Sample and Apply systems.

R008: WHEN v0.3.0 is prepared, THE SYSTEM SHALL remain compatible with v0.2 APIs/schema and pass release readiness checks without publishing.
