## Context

Add a feature named `schema` that enables std, the existing optional serde capability, and schemars. `RecipeManifest` remains available in all builds and derives serde only under `serde`; `schema` adds JsonSchema derives/exports for it and authoring definitions. Retain no schema dependency in default, alloc-only, or no_std configurations. Test structural schema output and update CI to exercise schema under both signal paths without weakening existing matrices.

## DoD

- Feature resolution is explicit and does not alter default feature behavior.
- Public Schema exports compile and describe the documented serializable types.
- Core no_std checks remain schema-free.
- CI tests the new feature matrix.

## Validation

- Schema-specific tests.
- Core f32 and i32 schema feature tests.
- Existing Rust 1.75 alloc/no_std checks.

## Escalate If

Schemars support cannot meet Rust 1.75 or forces schema dependencies into no_std builds.
