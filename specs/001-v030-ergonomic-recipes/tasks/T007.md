## Context

Add generic `WyrdRecipePlugin<R: Recipe>` and `WyrdRecipeInstance<R>` resource. Startup binds a recipe, resolves R::Ports, inserts it into WyrdWorld, and exposes the generational instance ID. Preserve WyrdPlugin's ordering and do not add systems that sample game components or apply game effects. Refactor `and_door` to use the typed recipe instance.

## DoD

- Bevy setup no longer hand-builds application port binding resources for the recipe.
- Removal/stale handles and resolve/bind failures are safe and contextual.
- Sample/Loom/Apply ownership remains explicit.

## Validation

- `cargo test -p wyrd-for-games-bevy`
- `cargo build -p wyrd-for-games-bevy --example and_door`
- Add lifecycle and typed binding integration tests.

## Escalate If

Bevy resource initialization requires hiding fallible binding failures or changes the existing system-set contract.
