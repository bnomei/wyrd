## Context

Add a dedicated exported `pattern!` macro. Its grammar is `id`, optional numeric path, `knots`, `exports { input name = knot.port; output name = knot.port; }`, and `threads`. It must lower to existing `PatternDef`/`Pattern` validation, preserve expression-once/source-order guarantees, and never weaken builder/pattern validation. Convert B01 to it.

## DoD

- `pattern!` returns `Result<Pattern, BuildError>` with documented public syntax.
- Macro supports aliases and valid exports; invalid and duplicate exports retain contextual errors.
- Tier B monostable uses the macro.
- Runtime, macro parity, and compile-fail diagnostics are covered.

## Validation

- `cargo test -p wyrd-for-games --test weave_macro`
- `cargo test -p wyrd-for-games --test weave_macro_ui`
- `cargo test -p wyrd-for-games --test patterns_cookbook`

## Escalate If

Pattern validation requires an error type change that would break v0.2 callers.
