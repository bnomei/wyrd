## Context

Implement `Weave::compose(id, |composer| ...)` with typed Bool/Level/Count wires, semantic helpers for cookbook-used operations, and a low-level generic knot/thread escape hatch that covers the complete existing catalog. It lowers exclusively through WeaveBuilder so owner, port, domain, Pattern, and DAG validation remain authoritative. Preserve explicit node identifiers and support branching/fan-out.

## DoD

- Generated topology is expressive without repeated output/input/connect temporaries.
- Fixed domain mismatches are prevented by typed helpers or return existing structured errors through generic operations.
- Composer output matches equivalent macro/builder Weaves.
- No typestate explosion or new Bevy dependency is introduced.

## Validation

- Focused composer parity, branching, domain, Pattern, and invalid-topology tests.
- `cargo test -p wyrd-for-games --test validate_edges`
- `cargo test -p wyrd-for-games --test coverage_authoring`

## Escalate If

Supporting a catalog operation would require silently changing its domain or port semantics.
