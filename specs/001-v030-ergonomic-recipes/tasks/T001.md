## Context

Use the existing `weave!` macro for fixed topology. Extract public `*_weave()` functions returning `Result<Weave, BuildError>` where cookbook recipes currently only expose `run_*`; retain every existing runner and its external behavior. Keep error-injection coverage out of the pedagogical topology where practical. Convert the Bevy example's graph to the macro but do not add recipe plugin behavior yet.

## DoD

- All fixed graphs in cookbook tiers A-D are declarative `weave!` definitions.
- Runners still demonstrate the same timing and output semantics.
- The Bevy example graph uses the same canonical syntax.
- Tests prove macro-built graphs preserve previous behavior.

## Validation

- `cargo test -p wyrd-for-games --test tutorial_ladder`
- `cargo test -p wyrd-for-games --test weave_macro`
- `cargo build -p wyrd-for-games-bevy --example and_door`

## Escalate If

The existing macro cannot express a current valid graph without extending public macro grammar; do not invent a second static DSL.
