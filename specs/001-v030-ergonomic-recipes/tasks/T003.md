## Context

Introduce public `Recipe`, `RecipeInstance<R>`, typed port resolution helpers/errors, and deterministic `RecipeManifest`. A Recipe constructs a Weave, resolves its associated Ports from a Runtime, has a default bind path, and exposes a manifest extracted from validated topology. Manifest includes weave ID, numeric path, SignalIn endpoints, SignalOut paths, and EmitCommand names. Manifest derives serde only under the existing optional `serde` feature; it remains available as a normal Rust value otherwise. Existing `Runtime`, `Weave`, and codec APIs remain unchanged.

## DoD

- Generic hosts can bind a Recipe and retain typed resolved ports.
- Missing or invalid required endpoints yield contextual Recipe errors.
- Equivalent weaves yield deterministic manifests.
- Contracts are exported from appropriate root and layer namespaces.

## Validation

- Focused new recipe/manifest tests.
- `cargo test -p wyrd-for-games --test coverage_authoring`
- `cargo test -p wyrd-for-games --test coverage_runtime`

## Escalate If

The required public trait cannot stay additive or a manifest cannot be derived without relying on macro source text.
